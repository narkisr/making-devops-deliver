Making Devops Deliver

Bringing back the DEV in Ops

Created by narkisr / @narkisr

The sound of BREAKAGE

  • OPS bottleneck
  • Progress/Change is hard
  • Collaboration sucks

Devops is => DEV!

Trust the source

  • Best focal point
  • Closest to the truth
  • Self documenting
  • Easy to collaborate
  • Use Git repos (its cheap)

Keep best practices

  • Testing, metrics, linting
  • Well defined build process
  • Continuous for everything (even for .vim)
  • Generate artifacts
  • Maintain versions/use dependency management

Be consistent

  • Same tools across the board
  • Well defined layout
  • A (sane) naming scheme

Evolve

  • Update regularly (OS, IDE, etc..)
  • Improve workflows/process
  • Follow online trends

Stay shy from

  • Manually maintained systems
  • Proprietary software
  • Single point of control
  • No clear security guidelines
  • Lacking Backup/restore
