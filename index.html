Source is the focal point

Best practices all across the board

  • Testing, quality metrics, linting
  • Have a build process
  • Have a continues for everything (yes your .vim)
  • Define meaningful artifacts
  • Maintain versions and use dependency management

Consistency

  • Using same tools across the board
  • Have a (sane) naming scheme and follow it

Modularity

  • Microservice your code, components should be just git repos
  • Monolithic Chef/Puppet repo is a smell

Warning signs

  • Manually maintained systems
  • Proprietary systems
  • Single point of control
  • No security considerations
  • No clear backup/restore plan
